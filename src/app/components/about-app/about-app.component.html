 User Story #2: As a user I want to add a record in the phonebook so that I can retrieve it.

Description

The user must have the ability to add a record on the phonebook.

The system must notify the user of any duplications, however the system must also allow duplicate records and also give the user the ability to merge duplicate records (Version 2)

Acceptance Criteria

  1. User can add a new record
  2. User gets notified of duplicated records
  3. User has an option to merge duplicate records
  4. - A save button is available on the screen for the user to save the contact
 User Story #3: As a user I want to delete a record so that I can free some space.

Description

Delete / Remove Contact

Acceptance Criteria

  1. Given that I have already selected a record from the search, then I must see a delete option.
  2. Given that I have selected the delete option then a message must be displayed requesting me to confirm my selection.
  3. Given that I have confirmed the delete action then the record must be removed from the phonebook and I am directed back to the list view
User Story #3: As a user I want to update a record so that I can capture the correct information.

Description

Edit Contact

Acceptance Criteria

  1. Given that I have already searched for a record, when I click the edit button a modal form must show with editable data.
  2. Given that I have already edited the details, when I click save then the updated details must be saved.
